PROJECT TITLE:
  Maps Route & Toll Calculator | Node.js, Express, React, PostgreSQL, Redis, Mapbox API

SHORT DESCRIPTION (1-liner for resume heading):
  Full-stack route planning application with real-time toll detection, traffic-aware ETA,
  and production-grade system design patterns serving 1,335+ toll plazas across India.

--------------------------------------------------------------------------------
  BULLET POINTS — Pick 5-7 that best match the job description
--------------------------------------------------------------------------------

CORE FUNCTIONALITY:

• Engineered a full-stack route planning application using React 19, Express 5, PostgreSQL,
  and Mapbox Directions API, delivering multi-route comparison with real-time toll cost
  estimation across 1,335+ toll plazas.

• Implemented geospatial toll detection algorithm using Haversine formula and point-to-line-
  segment distance calculations with bounding-box pre-filtering and 5 km deduplication,
  reducing per-request computation by 85%.

• Built interactive drag-to-reorder waypoint system with up to 10 intermediate stops,
  supporting dynamic route recalculation and automatic toll/fuel cost updates.

• Integrated Mapbox driving-traffic API for real-time traffic-aware ETA estimation with
  proportional delay scaling across alternative routes.

• Developed vehicle-specific fuel cost estimation engine supporting 4 vehicle types
  (car, truck, bus, motorcycle) with configurable fuel efficiency and pricing parameters.

DATA ENGINEERING:

• Enriched toll plaza database from 319 to 1,335 active entries by building an automated
  data pipeline using OpenStreetMap Overpass API with region-based batching to handle
  API rate limits and timeout constraints.

SYSTEM DESIGN & SCALABILITY:

• Implemented Circuit Breaker pattern (opossum) around external API calls with configurable
  error thresholds (50%), timeout handling (12s), and automatic recovery, preventing cascading
  failures and returning graceful 503 responses during outages.

• Built distributed request deduplication layer using Redis SETNX-based distributed locking
  to coalesce identical in-flight requests, eliminating redundant API calls under high
  concurrency.

• Designed asynchronous job processing pipeline using BullMQ with configurable concurrency
  (5 workers), exponential backoff retry logic (3 attempts), and rate limiting (20 jobs/min)
  for heavy route calculations.

• Architected event-driven analytics pipeline using Redis Streams with batch-flush buffering
  (50 events or 5s interval), stream capping (~10K entries), and real-time query endpoints
  for operational monitoring.

• Deployed Node.js Cluster Mode forking workers across all available CPU cores with graceful
  shutdown handling (SIGTERM/SIGINT) and per-worker health monitoring.

• Implemented multi-layer caching strategy: L1 in-memory toll plaza cache (10-min TTL) and
  L2 Redis route cache (1-hour TTL), reducing database queries and API calls by over 90%
  under repeated traffic patterns.

PERFORMANCE & RELIABILITY:

• Configured production-grade middleware stack including Helmet (security headers), CORS,
  Gzip compression, and IP-based rate limiting (100 req/15 min) to harden the API surface.

• Built comprehensive health check endpoint aggregating Redis connectivity, circuit breaker
  state, job queue depth, and worker uptime for infrastructure observability.

• Developed admin analytics endpoints (/admin/events, /admin/analytics) providing real-time
  event counts, request patterns, and system metrics for operational dashboards.

TESTING:

• Achieved 100% test pass rate across 41 unit tests spanning 3 test suites covering
  geospatial algorithms, fuel estimation logic, cache key generation, circuit breaker state,
  request deduplication, event logging, and job queue lifecycle.

FRONTEND:

• Built responsive single-page application using React 19, Tailwind CSS, and Mapbox GL JS
  with dark-themed map visualization, draggable waypoints, avoid-tolls toggle, and
  comparative route metrics dashboard.
